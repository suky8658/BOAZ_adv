# data

## Purpose
Stores cached JSON artefacts generated by the schema inspection and integrity validation pipeline. These files are consumed by `src/validator/integrity_loader.py` at agent runtime to avoid repeated database round-trips and LLM API calls.

## Key Files

| File | Description |
|------|-------------|
| `db_schema.json` | Full database schema snapshot: tables, columns, types, PKs, FKs, and LLM-generated descriptions with sample data |
| `db_integrity_result.json` | Merged integrity check results consumed by the agent's `get_table_details_tool` |

## For AI Agents

### Working In This Directory
- These files are **generated artefacts** — do not edit them manually.
- `db_schema.json` is produced by `src/db/db_schema.py` (`get_schema_info(use_llm=True)`); regenerating it costs OpenAI API tokens.
- `db_integrity_result.json` is produced by the integrity pipeline (physical + semantic managers) run from `test_agent.ipynb`.
- If either file is missing the agent tools will raise `FileNotFoundError` at runtime.

### Common Patterns
- Both files are read via `src/validator/integrity_loader.py` helper functions.
- The schema file includes `sample_data` arrays (10 rows per table) embedded alongside column metadata.
