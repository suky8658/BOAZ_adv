# gx

## Purpose
Great Expectations project directory. Holds the GE context configuration, expectation suites, and checkpoints used by the physical and semantic integrity validation pipeline (`src/db/integrity_physical.py`, `src/db/integrity_semantic.py`).

## Key Files

| File | Description |
|------|-------------|
| `great_expectations.yml` | Root GE context config — datasource, store, and data docs settings |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `checkpoints/` | GE checkpoint YAML files that wire expectation suites to validation actions |
| `expectations/` | Persisted expectation suite JSON files (auto-generated by GE) |
| `plugins/custom_data_docs/` | Custom CSS for the GE Data Docs HTML report |

## For AI Agents

### Working In This Directory
- The GE context is loaded at runtime via `gx.get_context()` inside the integrity manager classes — do not instantiate a second context.
- Checkpoint files (`integrity_checkpoint.yml`, `test_checkpoint.yml`) are created/updated programmatically via `context.add_or_update_checkpoint()`; editing YAML directly may conflict.
- Expectation suites in `expectations/` are regenerated on each validation run; the `.ge_store_backend_id` file is a GE internal marker — do not delete it.

### Common Patterns
- The PandasExecutionEngine + RuntimeDataConnector pattern is used throughout — data is passed as in-memory DataFrames, not read from files.
